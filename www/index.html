ProxyMusic

Author: Hervé Bitteur
Updated:
A new version of ProxyMusic has been released on 10-Apr-2007, to support MusicXML V2.0 (beta).

Project Goals

ProxyMusic provides a binding between Java objects in memory and an XML file using the MusicXML format. It should provide an easy way for Java programs dealing with music symbolic information (such as score scanners, music editors, music sequencers, etc) to read or write files in MusicXML. It is released under the GNU Public License version 2.

The binding is made available for any Java application as a dedicated jar file. There is one jar file per supported version of MusicXML:

  • MusicXML V1.1
  • MusicXML V2.0 (beta)

Requirements

JAXB2 is needed at run-time to perform marshalling or unmarshalling.

  • If you are using the latest Java version (Java 6 as recently released), JAXB2 is already included in the Java distribution.
  • If you are using Java 5, you will need to download a separate JAXB2 implementation, from the JAXB web site.
  • And since generics are used by the binding, you should forget about Java 1.4.

If ever you want to rebuild the binding by yourself, besides JAXB2, you'll need a tool to generate an XML schema from the original DTD files. JAXB2 is not currently able to do the job, but you can use a small utility called dtd2xs, free for non-commercial use, that is available on Syntext site.

Documentation

Use of the binding
A small example is available in two versions in the proxymusic.examples package. There are actually two examples HelloWorld_2_0.java and HelloWorld_1_1.java since there are slight differences in the user program implied by V1.1 and V2.0. This example performs marshalling and unmarshalling of the classical HelloWorld as available in the MusicXML tutorial.

A utility class proxymusic.util.Marshalling is available to simplify the handling of both marshalling and unmarshalling.

Regeneration of the binding
Note that the regeneration of the binding is not a mandatory task as the jar file from the download area can be used as is. A detailed recipy is however provided, that describes the proper sequence of steps (all the steps are now performed automatically).

In short, the binding is generated from the original set of DTD files found on Recordare web site. A few modifications are performed on the DTD files, the corresponding schema is generated and corrected and finally the various Java classes are generated. These steps are carried out automatically via an 'ant' build file (or via a NetBeans project). The resulting 300+ Java files use the JAXB2 binding for marshalling and unmarshalling. The comments found in the original DTD files are also copied into the proper Java files.

Current Status

ProxyMusic is available today in version 1.1 and version 2.0 beta to support the corresponding versions of MusicXML.

The modifications needed for MusicXML V2.0 have also been the opportunity to implement a more automated way for generating the various artifacts of the binding, which have been retrofitted in V1.1 as well.

Foreseen Contributions

This work started as a companion of Audiveris project, for which I needed a suitable binding to write MusicXML. After a few discussions on MusicXML forum, especially the help provided by Dominik Wei-Fieg, I developed this binding, documented the process, and made the result available as a project separate from Audiveris.

Through Audiveris, I use this binding for marshalling only, and my experience as an end-user is satisfactory. The unmarshalling aspect has not been thoroughly tested and would benefit from another project interested in unmarshalling MusicXML files to Java.
