ProxyMusic

Author: Hervé Bitteur
Updated:
ProxyMusic was released on 14-Jan-2007, under the GPL v2 license.

Project Goals

ProxyMusic provides a binding between Java objects in memory and an XML file using the MusicXML V1.1 format. It should provide an easy way for Java programs dealing with music information (such as score scanners, music editors, music sequencers, etc) to read or write files in MusicXML.

The binding is made available for any Java application as a dedicated jar file.

Requirements

JAXB2 is needed at run-time to perform marshalling or unmarshalling. If you are using the latest Java version (Java 6 as recently released), JAXB2 is already included in the Java distribution. If you are using Java 5, you will need to download a separate JAXB2 implementation (such as 2.0.4 or 2.1), from Jaxb web site. And since generics are used by the binding, you should forget about Java 1.4.

If you want to rebuild the binding, besides JAXB2, you'll need a tool to generate an XML schema from the original DTD files. JAXB2 is not currently able to do the job, but you can use a small utility called dtd2xs, free for non-commercial use, that is available on Syntext site.

Download

  • The download area contains the latest stable release. On this area you'll find:
    • The binding jar file proxymusic-1.1.jar. This is the only file you actually need to just use the binding.
    • If you are using an IDE such as NetBeans, you should be interested in feeding the IDE with two related archives: the javadoc proxymusic-1.1-doc.jar and the source proxymusic-1.1-src.jar.
    • If you want to regenerate the binding (for version V1.1 of MusicXML or for another one), here is the core material proxymusic-core.zip. Or if you prefer everything baked, get proxymusic-pack.zip.
  • The CVS repository can be accessed to retrieve any version of all software pieces.

Documentation

Use of the binding
A small example is available in the Test repository, performing marshalling and unmarshalling of the classical HelloWorld as available in the MusicXML tutorial.
Regeneration of the binding
A detailed recipy is provided, that describes the proper sequence of steps, manual and automatic. Note that the regeneration of the binding is not a mandatory task as the jar file from the download area can be used as is.

In short, the binding is generated from the original set of DTD files found on Recordare web site. A few manual modifications have to be done in the DTD files as described in the recipy, and the rest of the generation is performed automatically via an 'ant' build file (or via a NetBeans project). The resulting 300+ Java files use the JAXB2 binding for marshalling and unmarshalling. The comments found in the original DTD files are also copied into the proper Java files.

Current Status

ProxyMusic is available today in version 1.1, since it uses the MusicXML version 1.1.

When the new MusicXML version (V1.2) gets published, one can expect that a proxymusic version 1.2 will be easily generated.

Foreseen Contributions

This work started as a companion of Audiveris project, for which I needed a suitable binding to write MusicXML. After a few discussions on MusicXML forum, especially the help provided by Dominik Wei-Fieg, I developed this binding, documented the process, and made the result available as a project separate from Audiveris.

Through Audiveris, I use this binding for marshalling only, and my experience as an end-user is satisfactory. The unmarshalling aspect has not been thoroughly tested and would benefit from another project interested in unmarshalling MusicXML files to Java.
